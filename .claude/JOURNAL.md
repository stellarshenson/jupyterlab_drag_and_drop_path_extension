2. **Task [Extended] - Drag-and-drop path feature** (v0.8.4): Implemented file/folder drag-and-drop from the file browser into terminals, editors and notebooks as a hybrid extension<br>
   **Result**: Feature lets a dragged file or folder be dropped onto a terminal, Python file editor or notebook code cell to insert its path. Key architectural decision: converted the copier `kind: frontend` scaffold into a hybrid extension because relative paths in a terminal need the terminal's filesystem cwd, which the browser cannot inspect. Added a server extension - `handlers.py` exposes `api/drag-and-drop-path/server-info` (absolute server root) and `terminal-cwd/<name>` (cwd via `/proc` process-tree inspection ported verbatim from `jupyterlab_terminal_show_in_file_browser_extension`); `__init__.py` gained `_jupyter_server_extension_points`; `jupyter-config/server-config/` auto-enables it; `pyproject.toml` got the `jupyter_server` dependency, server-config shared-data and pytest config. Frontend split into `paths.ts` (pure, unit-tested path resolution, posix/pathlib formatting, shell escaping, drag-MIME parsing), `server.ts` (endpoint calls) and a rewritten `index.ts` that uses `ITerminalTracker`/`IEditorTracker`/`INotebookTracker` to attach Lumino `lm-dragover`/`lm-drop` handlers to each widget. Settings (`schema/plugin.json`): `pathType` (absolute/relative), `pythonPathStyle` (posix/pathlib), `pathlibConstructor`. Terminal paths are shell-escaped unquoted; Python paths are always quoted string literals or `pathlib.Path(...)` expressions; multi-file drags are no-ops. Verified the `application/x-jupyter-icontents` drag MIME carries a `string[]` against the installed JupyterLab. `build.yml` gained server-extension checks and `check_links` badge-URL ignores. Built via `make install`; both extension halves load OK, 23 Jest and 20 pytest tests pass, lint clean. Project version set to 0.8.4.

4. **Task - Fix tilde server root and terminal focus** (v0.8.4): Fixed relative paths resolving through a literal `~/workspace` and made terminal drops focus the terminal<br>
   **Result**: A relative-mode terminal drop produced `../../../~/workspace/kolomolo/.../nordea-demo-slides.md` instead of a simple relative path. Cause: JupyterHub sets the server `root_dir` to `~/workspace`, and `ServerInfoHandler` used `os.path.realpath`, which treats `~` as a literal directory name - from the server process cwd `/` that yielded `/~/workspace`, so the frontend `relative()` shared no prefix with the terminal cwd and walked all the way up to root. Fixed `handlers.py` to `os.path.realpath(os.path.expanduser(root_dir))` so the tilde expands to the real home directory. Also addressed terminal focus: the drag starts in the file browser which keeps focus, so `index.ts` now calls `app.shell.activateById(widget.id)` on a terminal drop (`app` added to `IExtensionState`). Rebuilt at 0.8.4; labextension OK, 23 Jest tests pass. Needs a server restart for the handler change and a browser reload for the frontend.

6. **Task - Pathlib `/` operator output** (v0.8.4): Switched the pathlib-style Python output to use pathlib's `/` operator instead of a single constructor call<br>
   **Result**: User requested the `pathlib` style render as idiomatic `Path` joins. Rewrote `formatForPython` in `src/paths.ts`: splits the path into segments, keeps the leading slash attached to the first segment for absolute paths (so the expression evaluates to the same absolute Path), and joins the remainder with `/`. Single-segment paths collapse back to a single constructor call. Examples: `/a/b/c` -> `pathlib.Path('/a') / 'b' / 'c'`, `a/b` -> `Path('a') / 'b'`, `/file` -> `Path('/file')`. Added two new Jest cases covering multi-segment join and single-segment collapse; the posix-style and constructor-choice tests are unchanged. 24 Jest + 20 pytest tests pass; rebuilt at 0.8.4 (`0.8.3 -> auto-increment`); labextension OK; lint clean.

9. **Task - Fix CI workflows** (v0.8.6): Pinned `webpack@5.106.2` in `resolutions` to sidestep a `license-webpack-plugin@2.3.21` crash that only fired on CI's fresh install<br>
